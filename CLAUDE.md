# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

A WebGL2 art piece simulating a faceted cola glass on a sunlit table, with photon-traced caustics, a raymarched glass render, and procedural cicada audio. Being turned into an on-chain generative gift series (plan: gift-code lazy minting on Base). No build system, no dependencies, no tests. Four files, loaded as plain scripts (no modules):

- `index.html` — markup, CSS, the slider UI, and the script tags
- `shaders.js` — all GLSL sources as JS template strings (kept in `.js` rather than `.glsl` files so the page works over `file://`, where `fetch()` is blocked)
- `main.js` — GL plumbing, UI reading, audio synthesis, and the render loop
- `haiku.js` — dev-only caption layer (generative haiku per deal, currently hidden via CSS). Loads last; hooks in via `window.onDeal` (fired by `randomize()`) and its own H-key listener. **Excluded from the minted artwork build** — main.js must never reference it, and the page must run cleanly without it.

There is no user-facing reroll: a viewer gets one deal per load (S saves a PNG; drag orbits).

**Contract**: `contract/` is a Foundry project (deps gitignored — `cd contract && forge install` after a fresh clone). `SummerGlasses.sol`: ERC-721 with gift-code lazy minting — `gift(claimAddr)` escrows payment against an ephemeral claim key's address; `redeem(claimAddr, to, sig)` mints when the claim key has signed `redeemDigest(to)` (raw digest, no EIP-191 prefix — signature binds the recipient, so redeems can't be front-run and anyone can relay; msg.sender gets the gas stipend); `reclaim` refunds the gifter after 365 days. Seed fixed at redemption from prevrandao+tokenId+recipient. `gift()` also mints the payer a keepsake from the companion `GiftReceipt.sol` (a separate, freely-transferable ERC-721 with a fully on-chain SVG card — project name, receipt no., gift date/time in UTC, a thank-you; carries no code and can't redeem anything). Its purpose is dual: an on-chain artifact of the gift, and defusing the wallet-simulator "you pay, get nothing" drainer heuristic that made MetaMask flag a bare `gift()`. Wiring is one-time and then immutable: `receipt.setMinter(glasses)` (only the glasses contract may mint receipts) + `glasses.setReceipt(receipt)`; both revert on a second call. The mint is guarded (`if(receipt != 0)`) so gifting still works with no receipt bound, and uses `_mint` not `_safeMint` so a receiver callback can never make `gift()` revert. Artwork stored via `SSTORE2.sol` chunks + three wrapper parts (from `dist/art-*.txt`); `tokenURI` reassembles the bootstrap with the token's seed, fully on-chain. `forge test` (44 tests across the `SummerGlasses` + `GiftReceipt` suites) includes a round-trip that writes `dist/.onchain-token.html` for browser checks. Deploy: `script/Deploy.s.sol` (env: PRICE/STIPEND/MAX_SUPPLY); freeze art manually at launch (see `LAUNCH.md` for the full runbook). ERC-2981 royalty via `setRoyalty` (default off). Optional marketplace thumbnails via `setImageBase` — mutable even after `freezeArt()` on purpose (the frozen artwork is the animation_url; the image is a convenience pointer, and `tokenURI` omits it while unset). Testing gotcha: helpers that call the contract (even views) consume a pending `vm.prank`/`vm.expectRevert` — precompute signatures first.

**Gift/redeem dApp**: `web/` — `gift.html` (wallet pays `gift()`, claim key generated in-browser, code shown before payment, printable card), `redeem.html` (code from `#k=` fragment or paste, fresh-wallet option, signs `redeemDigest(to)`, POSTs the relayer, reveals the artwork and hands over a permanent `view.html#<id>` link), and `view.html` (the gallery: every minted glass newest-first, lazy thumbnails, click for the live piece full-screen; `#<tokenId>` opens one, `#a=<address>` marks a collector's own). Shared logic in `web/lib/claim.js` (gift codes, keys, signing), `web/lib/rpc.js` (minimal JSON-RPC; all requests funnel through one page-wide gate — `MAX_INFLIGHT` 2, plus a `MIN_GAP_MS` 120 min spacing so we stay under Base's public per-second rate limit, not just its concurrency — that retries rate-limited calls with exponential backoff, because Base's public RPC throttles the gallery's owner+seed+artwork fan-out; `web/test/rpc-throttle.mjs`), `web/lib/token.js` (tokenURI/tokenHTML fetch/decode, `srcdoc` render, owner lookup; the artwork bytes are identical across glasses so the gallery fetches `tokenHTML` ONCE with a sentinel seed via `htmlForSeed`/`artTemplate` and splices each token's own seed in locally — N×60 KB reads collapse to 1×60 KB + N cached `seedOf` reads, proven byte-identical against live tokens), and `web/lib/thumbs.js`.

**Thumbnails**: the artwork's WebGL context has no `preserveDrawingBuffer`, so a canvas grabbed from outside the render task comes back blank (same reason the S-screenshot captures in-loop, `main.js:1031`). Instead each glass photographs itself: post it `{type:'summer-glass-snapshot-request', id}` and it replies `{type:'summer-glass-snapshot', id, jpeg}` from inside `frame()` after 15 frames (enough for the ~10-frame caustic accumulation). It reads pixels only — it consumes no randomness and cannot change a deal (regression-checked against a live token's seed). `thumbs.js` renders offscreen iframes 3 at a time (each live glass is a WebGL context; browsers cap those ~16) and caches results in IndexedDB by seed. Offscreen iframes are positioned off-viewport rather than `display:none` — a hidden iframe may never get a rAF tick and would never render. Address lookup scans `ownerOf` over the minted range rather than events — the collection isn't ERC721Enumerable and public RPCs cap `eth_getLogs` (Base's at 2000 blocks). Network coordinates in `web/config.js`, which exports each network by name (`anvil`, `baseSepolia`, `base`) plus a default that tracks whatever is being worked on — tests import the named block, never the default. **Gift codes** are five words and a five-digit number (`amber cicada glass noon shade 40317`), not a raw key: the words+number ARE the secret and the secp256k1 private key is derived from them with PBKDF2 (250k rounds, ~85ms). That slowness is load-bearing — a gift's claim address is public on-chain from the moment it is paid for, so guessing is an offline attack with nothing to rate-limit it; 5 words from BIP-39's 2048 (55 bits) + 5 digits (16.6 bits) ≈ 71.6 bits, times ~2^18 work per guess. **Do not shorten the format without redoing that arithmetic.** Parsing is forgiving (case, dashes, stray punctuation) and also rescues a single-character slip per word — a non-word token snaps to the UNIQUE BIP-39 word one edit away (so a card whose "job" is read as "iob" — by OCR or by a person retyping it — still redeems), never when the nearest word is ambiguous; `readCode()` reports what was corrected and `badWords()` names only the truly unrecoverable ones (`web/test/claim-ocr.mjs`). BIP-39 was chosen because no two of its words share a 4-letter prefix. Pre-v2 base58 codes still decode (`decodeLegacyCode`) so gifts sold under the old format stay redeemable. Crypto is vendored audited single-files (`web/vendor/`: noble-secp256k1, js-sha3, BIP-39 wordlist) — signatures are RFC6979-deterministic and byte-identical to cast (`web/test/claim-parity.mjs` proves it). `relayer/relayer.js` (plain Node, no npm deps) submits redeems via `cast` child-processes: simulate-then-send, nonce-safe queue, per-IP rate limit, self-repaying via the on-chain stipend. **Production relayer is `web/api/redeem.mjs` + `status.mjs` (shared logic in `web/lib/relayer-core.mjs`, kept out of `api/` because Vercel makes every file there its own function)** — the same logic as a Vercel serverless function using viem instead of `cast`, so the site and the redeem endpoint ship together on one domain with nothing to host (`config.js` points `relayer:'/api'`). Env: RELAYER_PK/CONTRACT/RPC_URL/CHAIN_ID. `relayer/relayer.js` stays for anvil + the test harnesses. The viem function is proven against live Base Sepolia by `web/test/relayer-viem-sepolia.mjs` (needs a throwaway key funded via `web/.test-key`); it minted token #10 with the on-chain seed matching the function's return byte-for-byte. Both the function and its Sepolia test learned the same lesson: Base's public RPC pool is load-balanced, so a read right after a write can hit a node a block behind — the function retries "unknown gift" through a backoff before calling a gift spent, and the test retries its verification reads. **Redeem must never lie**: a mint can land while the reply is lost (relayer dies mid-flight, second click on a consumed gift), so on any relayer error the page re-reads the chain — an empty gift slot means it worked, and it finds the token and reveals it instead of reporting failure. Unreachable-relayer, already-redeemed, rate-limit and bad-signature errors each get their own plain-language explanation, and a `/status` preflight warns before someone fills in an address. Covered by `node web/test/redeem-recovery.mjs`, which drives the real page in headless Chrome against a stub relayer that can be made to vanish or to lie after a successful mint. Full-stack test: `node web/test/e2e-anvil.mjs` (spawns anvil + deploy + relayer; `KEEP=1` leaves the stack up and prints an un-redeemed code for browser testing). Gotcha: do NOT add a `{"type":"module"}` package.json in `web/` — it breaks Node's `require()` of the UMD sha3 vendor in the test harnesses.

**Minting build**: `node build.js` (plain Node, no deps) produces `dist/` (gitignored): `summer-glasses.html` (the piece as one self-contained file, haiku/legend stripped — open directly to check a build), `payload.b64` (base64 of the gzipped single file, the bytes the contract stores in SSTORE2 chunks), and `bootstrap[-template].html` (the HTML a `tokenURI` emits: sets `window.TOKEN_HASH`, inflates the payload via `DecompressionStream`, `document.write`s it — the window survives `document.open()`, so main.js sees the hash). Builds are byte-reproducible (gzip without mtime). ~43 KB gzipped / 3 chunks as of 2026-07.

**Deterministic seeding**: every load deals one glass from a single entropy string — `window.TOKEN_HASH` if a minting bootstrap injected it (this also disables `#randtest`), else `#seed=<hex>` from the URL, else fresh randomness (the chosen seed is logged to the console with a reproduction URL). The string is folded through `xmur3` into the `mulberry32` seed that drives `randomize()`; same hash ⇒ same deal, verified down to every control value. Per-frame photon sampling and audio remain `Math.random()` on purpose — temporal dither and ambience, not identity. After each deal `window.$features` holds a machine-readable summary (drink, glassware, pattern, time of day, canopy, table, ice) for indexers. Determinism is tested by injecting a serializer page (`.det-test.html`, generated, gitignored) that dumps the full deal state into `document.title` under headless Chrome.

`glass-caustics-sim_33.html` is the original single-file version, kept as a reference/backup; don't edit it.

**Run it:** open `index.html` in a browser (`open index.html`). Requires WebGL2 with `EXT_color_buffer_float`. Debug via the browser console — shader compile errors are thrown with line-numbered source.

## Architecture

The critical design idea: **one shared glass model drives every pass**, so the photon tracing, the visible render, and the shadows all agree about the glass geometry.

- **`SHARED`** (GLSL string in `shaders.js`): the glass model — a **parameterized surface of revolution**: `profileR(y)` is a Catmull-Rom spline through 8 radius knots (`u_prof[8]`, spanning `u_y0..u_H`) with an analytic stem+foot below `u_y0` for stemware. On top of it: the pattern height field (`facetPattern`, dispatched by `u_pat`: diamond/ribs/spiral/hobnail), `outerRadius`/`outerNormal`, wall thickness (`wallAt`), inner-surface helpers (`innerR`, `innerNormal`, `innerChord` — iterated cylinder chord against the revolved profile), condensation, wall bubbles (`bubbleAt`), and the 3 lights with leaf-dapple masks (`dapple`). It is string-interpolated into **both** the photon vertex shader and the composite fragment shader. Any change to the glass shape must go here, or the caustics and the rendered glass will disagree.

- **Preset tables in `main.js`** drive the generative diversity: `SHAPES` (radius knots + stem/foot per glass family), `TIMES` (full light environments: sun angle/colour, sky/leaf/ground palette, penumbra; `night` preset = string-light bulbs + stars instead of a sun), `LIQUIDS` (absorption hex, turbidity, fizz, refractive index, scatter colour). The `randomize()` function deals coherent combinations (shape→liquid compatibility, ice only in tumblers with a ≥50% cold pour) by **setting the DOM controls** — `frame()` reads controls every frame, so there is no parallel state, except a few seed variables (`spillSeed`, `canRot`, `tabRot`, and `glassN` — the glass refractive index, raised to ~1.55–1.58 on "crystal" deals that also get water-clear tint, high dispersion, and deep cuts) rolled per deal. `#randtest` in the URL hash hammers it 60× for smoke-testing.

- **Environment**: `canopyField()` in SHARED defines what hangs overhead (broadleaf / fine leaf / palm / pergola / parasol via `u_canopy`); the dapple and the visible sky canopy both read it, so shade always matches reflections. Palm/parasol anchor to `u_canopyC` (the glass projected up the sun ray) so their shade lands on the glass. Table surfaces (stone / dark wood / tiles / concrete / pale wood via `u_table` 0–4) are procedural albedo in the COMP table branch. Caustic exposure is auto-compensated in `main.js`: window-area dilution × expected liquid/mist absorption (capped 5×) × expected-table-luminance contrast tracking (0.85–2.2×, anchored at the golden-afternoon/stone default; combined absorption+contrast product capped 6×). The photon shader applies a matching hue-preserving tint floor (luminance 0.075) so deep-coloured drinks keep a luminous caustic in their own hue — the JS estimate mirrors that floor.

- **`PHOTON_VS`**: attribute-less vertex shader; each of the 512×288×3 vertices (`gl_VertexID`) is one photon from one light, refracted through entry facet → wall → interior (cola with Beer–Lambert absorption, or air) → far wall → exit facet, landing as a point splat on the table. Runs twice per frame: `u_mode 0` = transmitted caustic, `u_mode 1` = Fresnel-reflected arc. Per-photon wavelength gives dispersion. `kill()` discards a photon by moving it off-clip.

- **Render pipeline per frame** (in `frame()`): the previous frame's 1024² RGBA16F caustic buffer is copied with ~0.90 decay into its ping-pong partner (`FADE_FS`), then this frame's photons are additively splatted on top — a rolling accumulation of ~10 frames that denoises the caustic; the photon hashes take a per-frame `u_seed` so each frame samples new photons. Then: separable Gaussian blur into a 512² buffer (soft halo) → `COMP_FS` full-screen composite (raymarches the glass SDF `sdGlass`, shades cola/table/courtyard `envColor`, samples both caustic textures) into a full-res HDR scene target → bloom (`BRIGHT_FS` threshold at quarter res, blurred with `BLUR_FS`) → `FINAL_FS` adds bloom and applies the camera artifacts (handheld wobble, corner chromatic aberration, far-field depth of field) plus vignette/tonemap/grain to screen. `COMP_FS` outputs linear HDR with the hit distance in alpha (the DoF pass depends on that); all finishing lives in `FINAL_FS`.

- **Caustic texture mapping**: photons splat into a table window mapped by `clip = (hit.xz - u_caustC)/u_caustS`; the composite reverses this exact mapping when sampling `u_caust`. Both sides use the same shared uniforms (computed in `main.js` from the main sun direction — a low sun gets a longer window), so they stay in sync by construction.

- **JS side** (`main.js`): UI sliders are read every frame and pushed as uniforms (`setShared` for values needed by both programs). Color pickers become absorption spectra via Beer–Lambert (`sigFrom`). Lights sway with a procedural wind/gust system (`snz` 1D noise) shared with the audio. `buildAudio()` synthesizes cicadas with Web Audio (band-passed noise gated by ~112 Hz click trains); the leaf-rustle gain is driven from the render loop's gust envelope.

## Conventions

- `shaders.js` and `main.js` are classic scripts sharing top-level globals; `shaders.js` must load first (script order in `index.html`).
- GLSL uniforms are prefixed `u_`; uniform locations are fetched lazily via a Proxy (`U(prog)`), so a typo'd uniform name fails silently — check names carefully.
- Comments in the shaders explain physical intent (why, not what); keep that style.
